# Cardboard — вёрстка карточек товара

Standalone-инструмент семейства Content Pulse (как Rate Setter). Локальное desktop-приложение: все данные остаются на компьютере.

## Запуск

**Desktop (основной режим):**
```bash
cd cardboard-standalone && python3 main.py
```
Зависимости (pywebview, Pillow) доустановятся сами при первом запуске.

**Браузер (фолбэк):** открыть `index.html` в Chrome. Отличие: фото встраиваются в файл проекта (миниатюрами), путей нет, автосохранение — в localStorage.

## Проект — как сессия Capture One

При запуске — окно проектов: список недавних (открываются в один клик) или новый. Без проекта работа не начнётся — это гарантия, что ничего не потеряется.

Название нового проекта собирается из полей **Бренд + Съёмка + Дата** (формат `BRANDX_название_2026-07-06`, пустые части выпадают) — или вписывается руками.

Проект = **папка**: внутри файл `имя.cardboard` (JSON: пути к фото, структура, шаблоны) и `previews/` — локальные JPEG-превью до 1200px. Превью пишутся один раз при импорте, дальше проект открывается мгновенно и показывает фото даже если исходники недоступны. **Автосохранение** после каждого изменения. ⌘Z / ⇧⌘Z — отмена и возврат.

**При закрытии** приложение спрашивает про несохранённое и предлагает удалить файл проекта, если он больше не нужен.

## Что делает

1. **Закидываешь фото** — drag-and-drop, «Импорт фото» (⌘I), вся папка разом (с окном прогресса), или слежение за папкой. Фото не копируются: приложение запоминает пути + держит превью в папке проекта. Вкладки галереи: Фото и Референсы (метка REF).
2. **Собираешь карточки** — «+ Карточка» → шаблон → фото из галереи в слоты перетаскиванием. Кнопка «Шаблон» на карточке: заменить шаблон (фото останутся по порядку) или отредактировать его прямо из карточки. Цвет фона карточки — пикер в шапке.
3. **Шаблоны** — канвас A4 (Г/В), заглавное фото, слотов в ряду, пропорции слотов (или «авто» — по фото), имена слотов (Фронт / Спинка / Деталь). Карточка рендерится ровно как превью шаблона. Библиотека хранится локально и внутри проекта.
4. **Структурируешь доп. контент** — свободные контейнеры: порядок фото (S/M/L), комментарий, свой цвет фона.
5. **Выгружаешь:**
   - **Экспорт PDF (⌘E)** — многостраничный PDF-файл: A4 300dpi, карточка = страница один в один, фото из оригиналов; галочка «убирать пустые слоты» оставляет чистый лист;
   - **Печать** — тот же макет через системный диалог;
   - **Список** — CSV (для Excel) + TXT: карточка → слот → файл → **полный путь**.

## Если фото переехали

Файл → **«Подставить папку с фото» (Locate)** — выбираешь папку (подпапки сканируются), фото сопоставляются по имени файла, пути обновляются, превью перечитываются. При открытии проекта с пропавшими файлами приложение само предложит это сделать. Сценарий: макет собран по превью, чистовые фото довыгружены позже в другую папку.

Если macOS блокирует папку (Загрузки/Документы/Рабочий стол) — приложение покажет системный запрос доступа или предложит выбрать папку в диалоге.

## Интерфейс

- Вкладки Карточки / Доп. контент закреплены при прокрутке.
- Боковые панели сворачиваются полосами «Галерея» и «Навигация» по краям окна — всё поле под работу.
- Ширина галереи тянется за разделитель.

## Синхронизация с Capture One (хоткей)

1. В Capture One создать экспорт-рецепт (JPEG, папка, например `~/Pictures/cardboard-inbox`) и назначить хоткей на «Export with recipe».
2. В Cardboard — «Следить за папкой», выбрать её.
3. Выделил фото → хоткей → через 2 секунды фото в галерее Cardboard.

## Сборка .app (macOS)

```bash
cd cardboard-standalone
/Library/Frameworks/Python.framework/Versions/3.14/bin/python3 -m PyInstaller Cardboard.spec --noconfirm
open dist/Cardboard.app
```

Нюансы, чтобы не наступать снова:
- В .app-бандле PyInstaller кладёт данные в Resources и делает симлинк из Frameworks — WKWebView не грузит file:// по симлинку (белое окно). Поэтому `find_frontend()` в main.py делает `.resolve()`.
- `private_mode=False` + `storage_path` (~/Library/Application Support/Cardboard), иначе localStorage с шаблонами стирается при каждом запуске.
- В Info.plist — usage-описания папок (Загрузки и т.д.), иначе macOS молча запрещает чтение при открытии проекта без диалога.

Самопроверка сборки: `CB_DIAG=1 ./dist/Cardboard.app/Contents/MacOS/Cardboard` — печатает READY/TOPBAR/BRIDGE и выходит.

## Технически

- `main.py` — pywebview-мост: диалоги, превью (Pillow: RAM + дисковый кэш previews/), слежение за папкой, Locate, сохранение проекта, сборка PDF (Pillow, A4 300dpi), диалоги при закрытии.
- `index.html` — весь UI, чистый JS без фреймворков. Dual-mode: desktop (пути) / браузер (встраивание).
- Префикс функций: `cb*`. Модель: `App.photos / cards / containers / templates`.

## Проверка

- Python-юниты (headless): превью и дисковый кэш, PDF, Locate-скан, фильтры импорта, слежение, сохранение.
