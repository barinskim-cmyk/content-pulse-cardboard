# Cardboard — установка на macOS

Cardboard распространяется напрямую (не через App Store) и пока не подписан сертификатом Apple, поэтому при первом запуске macOS перестрахуется и предупредит. Это штатное поведение системы для любых программ «не из App Store», а не признак проблемы. Ниже — как открыть приложение за минуту.

## Скачать

Всегда свежая сборка: **https://github.com/barinskim-cmyk/content-pulse-cardboard/releases/latest** — файл `Cardboard-macOS.zip`. Сборку делает робот GitHub из открытого исходного кода (см. «проверка безопасности» ниже).

## Установка

1. Скачайте `Cardboard-macOS.zip` и распакуйте (двойной клик).
2. Перетащите `Cardboard.app` в папку **Программы** (Applications).

## Первый запуск: разрешить программе открыться

**macOS Sequoia (15) и новее:**

1. Двойной клик по Cardboard — появится «Приложение не удалось проверить». Нажмите **Готово** (не «Переместить в Корзину»!).
2. Откройте **Системные настройки → Конфиденциальность и безопасность**, прокрутите вниз.
3. Возле строки «Использование Cardboard заблокировано» нажмите **Открыть всё равно** и подтвердите паролем.
4. Это нужно один раз — дальше приложение открывается как обычно.

**macOS Sonoma (14) и старее:**

1. Правый клик (или Ctrl+клик) по Cardboard.app → **Открыть**.
2. В окне предупреждения снова нажмите **Открыть**.

**Если macOS пишет «приложение повреждено»** — это карантинная метка на файле, скачанном из интернета. Откройте Терминал и выполните:

```bash
xattr -cr /Applications/Cardboard.app
```

После этого приложение откроется обычным двойным кликом.

## Доступ к папкам с фото

При первом открытии проекта, чьи фото лежат в Загрузках, Документах или на Рабочем столе, macOS спросит: «Cardboard хочет получить доступ к папке...» — нажмите **Разрешить**. Если случайно запретили: Системные настройки → Конфиденциальность и безопасность → Файлы и папки → Cardboard → включить нужные папки.

## Как убедиться, что файл не подменён (проверка безопасности)

Три независимых уровня, на выбор:

**1. Происхождение сборки (аттестация GitHub).** Приложение собирает не человек на своём ноутбуке, а робот GitHub из исходного кода, который открыт и виден всем в репозитории. GitHub криптографически подтверждает: «этот файл собран из вот этих исходников». Проверить (нужен [GitHub CLI](https://cli.github.com)):

```bash
gh attestation verify ~/Downloads/Cardboard-macOS.zip -R barinskim-cmyk/content-pulse-cardboard
```

**2. Контрольная сумма.** Рядом с архивом в релизе лежит `Cardboard-macOS.zip.sha256.txt`. Сверьте:

```bash
shasum -a 256 ~/Downloads/Cardboard-macOS.zip
```

Строка должна совпасть с содержимым `.sha256.txt` символ в символ.

**3. VirusTotal** (virustotal.com, принадлежит Google): перетащите zip на страницу — файл проверят ~70 антивирусов сразу.

## Что приложение делает с данными

Всё локально: фото не копируются и никуда не загружаются, проект — папка с файлом `.cardboard` и превью на вашем диске. Отчёты об ошибках отправляются только если вы явно разрешили это при первом запуске (анонимно, без путей и имён файлов; выключается в Справке).
